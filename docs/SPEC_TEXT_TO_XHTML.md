# 青空文庫テキスト形式からXHTMLへの変換仕様書 (UTF-8版)

本ドキュメントは、青空文庫形式のテキストファイルをUTF-8エンコーディングで読み込み、UTF-8エンコーディングのXHTML 1.1ファイルへ変換するプログラムの仕様を規定します。

## 1. 概要

本ツールは、UTF-8で保存された青空文庫形式のソーステキストを解析し、注記（ルビ、アクセント、文字修飾、構造指示など）を適切なXHTMLタグに変換して出力します。

## 2. 入出力形式

### 2.1 入力テキスト

- **エンコーディング**: UTF-8
- **改行コード**: CRLF (`\r\n`) 推奨
- **構成**:
  - **ヘッダ**: ファイル先頭から最初の空行まで。タイトル、著者名などが含まれる。
  - **本文**: メインコンテンツ。
  - **後記（地本情報）**: 「底本：」という行以降の書誌情報。

### 2.2 出力ファイル

- **形式**: XHTML 1.1
- **エンコーディング**: UTF-8
- **メディアタイプ**: `application/xhtml+xml`

## 3. 解析ロジック

### 3.1 全般的な挙動

- 入力は文字単位（またはヘッダの行単位）で読み込まれます。
- UTF-8の全範囲の文字を許容します。
- 旧仕様にあった「JIS外字警告」や「半角カタカナ警告」は、UTF-8環境においては原則として不要ですが、青空文庫の標準的な表記ルールから外れる文字については警告を出すことが望ましいです。

### 3.2 ヘッダ処理

最初の空行までを行単位で読み込み、メタデータを抽出します。

- **1行目**: 作品名（タイトル）
- **2行目**: 著者名（「著者名」または「編者名」「訳者名」など）
- **以降の行**: 副題、原作者、翻訳者などが含まれる場合があります。接尾辞（「訳」「編」等）を基に種別を判定します。

**出力への反映**:

- `<title>` タグに「作品名（著者名）」の形式でセット。
- `<div class="metadata">` 内にメタデータテーブルを作成。
- `<meta>` タグ（DC.Title, DC.Creator, DC.Publisher="青空文庫"）を出力。

### 3.3 本文処理

文字単位で走査し、以下のトリガー文字を検出した際に特定の解析処理へ移行します。

| トリガー | 名称           | 内容                                                 |
| -------- | -------------- | ---------------------------------------------------- |
| `［`     | 注記開始       | `［＃...］` 形式のコマンド解析を開始する。           |
| `〔`     | アクセント開始 | `〔...〕` 形式の欧米語アクセント解析を開始する。     |
| `《`     | ルビ開始       | 直前の文字列に対するルビ解析を開始する。             |
| `｜`     | ルビ区切り     | ルビを振る対象文字列の開始位置を明示する。           |
| `底`     | 後記チェック   | 「底本：」の文字列を検出し、後記モードへ切り替える。 |

#### 標準テキストの処理:

- HTML特殊文字（`<`, `>`, `&` 等）はエスケープします。
- 改行は `<br />` に変換します（ブロック要素内を除く）。

### 3.4 注記コマンド解析 (`［＃...］`)

注記コマンドは再帰的に解析され、ブロック要素またはインライン要素として出力されます。

#### ブロックコマンド（開始・終了指示）

- **字下げ・地付き**: `［＃ここから...字下げ］` → `<div class="jisage_N">`
- **罫囲み**: `［＃ここから罫囲み］` → `<div class="keigakomi">`
- **終了指定**: `［＃...終わり］` または `［＃ここで終わり］` で対応する `</div>` を出力。

#### インラインコマンド

- **見出し**: `［＃大見出し］` → `<h3 class="o-midashi">` 等。
- **文字装飾**:
  - `太字` → `<span class="futoji">`
  - `斜体` → `<span class="shatai">`
  - `傍点` → `<em class="sesame_dot">`
- **画像引用**: `［＃（...）入る］` → `<img>` タグを出力（※パスの解決のみ行い、外字画像化処理は行わない）。

### 3.5 ルビ処理

- **形式**: `対象文字列《ルビ》`
- **出力**: `<ruby><rb>対象文字列</rb><rp>（</rp><rt>ルビ</rt><rp>）</rp></ruby>`
- `｜` による境界明示がある場合は、それに従います。

### 3.6 外字処理（UTF-8版簡略化）

以前の仕様に含まれていた「外字を画像ファイルへ置換する処理」は**行いません**。

- **外字注記**: `※［＃...］` 形式の記述は、以下の通り処理します。
  - 記述内容をそのままテキストとして出力するか、もしくは `<span>` タグで囲み `class="notes"` を付与して出力します。
  - UTF-8で直接表現可能な文字（JIS X 0213文字等）が注記されている場合は、可能であればその文字自体を出力します。

### 3.7 出力構成

以下の構造を持つXHTMLファイルを作成します。

1. **XML宣言**: `<?xml version="1.0" encoding="UTF-8"?>`
2. **DOCTYPE**: XHTML 1.1
3. **HTML構造**:
   - `<head>`: タイトル、CSSリンク (`aozora.css`)、UTF-8指定のmetaタグ。
   - `<body>`:
     - `<div class="metadata">`: 作品情報。
     - `<div class="main_text">`: 本文。
     - `<div class="bibliographical_information">`: 底本情報。
     - `<div class="notation_notes">`: 表記に関する注記（自動生成）。

## 4. エラー処理

- **例外**: 閉じタグの不足、不正な入れ子構造などを検出した場合はエラーを発生させ、処理を中断またはスキップします。
- **警告**: 解析不能なアクセント記号や、推奨されない文字種を検出した場合は標準エラー出力に警告を表示します。
